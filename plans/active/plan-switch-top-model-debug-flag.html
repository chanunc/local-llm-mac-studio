Design pillars

🔇 Zero cost when off

Module-level DEBUG + dbg() → stderr only. Without the flag: byte-identical stdout, same commands, same exit codes. No new dependencies.

🔑 Redaction by construction

Headers / api_key never printed — always ***redacted***. Repo is publicly mirrored; a pasted debug transcript must be safe.

🪵 No second registry

Log path derived from the > /tmp/….log redirect already in each recipe's start_cmd — nothing new to maintain.

🚨 Log-tail on timeout: always-on

Proposed default (not debug-gated): timeout is the one place something is known-wrong; the last 12 log lines are the only evidence.

Verification

1 · dry-run --debug: stderr trace, stdout diff-identical
2 · no flag → byte-identical to current (regression guard)
3 · forced failure → log tail fires on timeout
4 · clean live run → timings + redaction + PASS trace
5 · grep debug output for live key → must be absent

Verdict: justified. High-frequency failure class (your own 2026-06-11 triage log), all evidence currently discarded, ~30–40 lines behind a flag, one redaction rule. Completes the script's contract: --dry-run shows what would run; --debug shows what did run and what came back.
