PEFT / LoRA

Fine-tune behavior without copying the whole model.

PEFT means parameter-efficient fine-tuning. LoRA is the common method: freeze the base model, train tiny adapter weights, then switch or merge those adapters depending on the experiment.

Mental model: the base model gives capability and speed. The LoRA adapter nudges behavior: style, workflow rules, JSON format, terminology, or translation preference.

Visual: What PEFT Does

PEFT means Parameter-Efficient Fine-Tuning. It is the broader idea: train only a small set of added or selected parameters while the original model stays mostly frozen. LoRA is one PEFT method.

For your experiments: PEFT is the strategy, LoRA is the adapter type, and quantization is the later step that helps MacBook or Android fit.

Visual: What LoRA Adds

LoRA means Low-Rank Adaptation. A normal layer has a big weight matrix W. LoRA keeps W frozen and learns two small matrices, commonly called A and B. The adapter update is added to the original layer during inference.

In shorthand: output uses W + B x A. The expensive base model stays fixed; the small adapter learns the task-specific change.

Normal layer y = xW

The input x passes through the frozen base weight W.

LoRA layer y = xW + xBA x scale

The adapter adds a small learned direction on top of the base output.

Scale scale = alpha / r

r is rank. alpha controls adapter strength.

The key trick is choosing a small rank r. If a layer is 4096 x 4096, a full update has about 16.8M parameters. With r = 8, LoRA trains about 65K parameters for that layer: 4096 x 8 + 8 x 4096.

Small Code Example

from transformers import AutoModelForCausalLM, AutoTokenizer
from peft import LoraConfig, TaskType
from trl import SFTConfig, SFTTrainer
from datasets import load_dataset

base_model = "Qwen/Qwen3-0.6B-Instruct"
dataset_file = "datasets/macstu-workflow.jsonl"

# Dataset link: swap this file for android-automation.jsonl or thai-style.jsonl
train_dataset = load_dataset(
    "json",
    data_files=dataset_file,
    split="train",
)

tokenizer = AutoTokenizer.from_pretrained(base_model)
model = AutoModelForCausalLM.from_pretrained(base_model)

lora = LoraConfig(
    task_type=TaskType.CAUSAL_LM,
    r=8,                  # adapter rank: smaller = lighter
    lora_alpha=16,        # adapter scaling
    lora_dropout=0.05,
    target_modules="all-linear",
)

args = SFTConfig(
    output_dir="out/macstu-workflow-lora",
    max_length=1024,
    per_device_train_batch_size=1,
    gradient_accumulation_steps=8,
    learning_rate=1e-4,
    num_train_epochs=1,
)

trainer = SFTTrainer(
    model=model,
    args=args,
    train_dataset=train_dataset,
    processing_class=tokenizer,
    peft_config=lora,
)

trainer.train()
trainer.save_model("adapters/macstu-workflow-lora")

Adapter Switching

from transformers import AutoModelForCausalLM

model = AutoModelForCausalLM.from_pretrained(
    "Qwen/Qwen3-0.6B-Instruct"
)

model.load_adapter(
    "adapters/macstu-workflow-lora",
    adapter_name="macstu"
)
model.load_adapter(
    "adapters/thai-style-lora",
    adapter_name="thai"
)

model.set_adapter("macstu")
# generate Mac Studio workflow answer

model.set_adapter("thai")
# generate Thai translation

model.disable_adapters()
# generate with the untouched base model

Your Experiment Flow

1. Choose base Pick a model that already fits the target device.
2. Train LoRA Use a focused dataset: workflow, Android JSON, or Thai style.
3. Compare Test base vs adapter on the same held-out prompts.
4. Merge Merge the winning adapter when deployment needs one model file.
5. Quantize Compress for MacBook, Android, or local serving runtime.

Mac Studio Workflow

  • Teach repo-specific commands and caveats.
  • Optimize for short operational answers.
  • Evaluate with held-out workflow questions.

Android Automation

  • Use a tiny base model.
  • Train strict JSON action output.
  • Merge and quantize before device testing.

Thai Translation Style

  • Train tone and terminology preference.
  • Preserve commands, paths, ports, and model IDs.
  • Compare against base and ChindaMT output.

One Rule To Remember

LoRA changes behavior. Quantization changes size. Runtime changes speed. A successful PEFT experiment keeps those three responsibilities separate.
