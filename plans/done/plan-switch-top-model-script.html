🧬 Design pillars

Dynamic ranking, static recipes

Top-5 always re-parsed from the markdown. LAUNCH_RECIPES keyed by model-name substring carries what the table can't: GGUF paths, lms keys, parser flags. Unknown new model → listed as [no launch recipe], ~6-line dict entry to enable.

Maximum reuse

  • chk_llm_macstu.py → probe / identify / loaded-model
  • switch_opencode_config.py → IP + config write
  • bench_api_tool_call.py → read_file tool fixture
  • Start/stop commands verbatim from CLAUDE.md

Guards

  • Disk check before any server is stopped
  • lm-studio guardrail off→load→high dance
  • Key collisions pinned with --identifier
  • No run-state written to any doc (Sync Policy)
